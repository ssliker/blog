1.从工作区到版本库：
    1.提交至工作区：
        在目录中执行git init之后，该目录称为git仓库的工作区，放在工作区的所有文件会被git监控；

    2.提交至暂存区：
        1.只能从工作区提交至暂存区；
        2.提交命令：git add 文件或者目录

    3.提交至版本库：
        1.只能从暂存区提交至版本库；
        2.提交命令：git commit -m '提交说明信息'

2.操作记录查看：
    1.查看git仓库状态：
        1.会显示当前仓库中文件的状态信息，并给出处理的提示信息；
        2.命令：git status
    
    2.比对暂存区/版本库和工作区文件区别：
        1.显示暂存区或者版本库和工作区文件内容差异，如果暂存区和工作区有差异则优先显示暂存区和工作区差异，否则显示版本库和工作区差异；
        2.命令：git diff
    
    3.查看工作区和版本库的区别：
        1.命令：git diff HEAD -- 文件/目录
    
    4.查看版本库提交历史记录：
        1.详细显示版本库中各个版本的提交记录；会显示各个版本的id，提交说明，提交作者，提交日期等信息；该命令显示的内容会随着回退操作而改变；
        2.命令：git log
    
    5.查看版本操作历史记录：
        1.详细显示版本提交以及回退操作的历史记录，相比git log多了回退操作的记录：
        2.命令：git reflog

3.版本回退：
    1.逐级回退：
        git reset --hard HEAD^    回退到上一级
        git reset --hard HEAD^^   回退到上上一级
        git reset --hard HEAD^^^  回退到上上上一级
        上述操作根据HEAD后面的^确定回退的级数，并且只能回退；
    2.通过版本号回退：
        git reset --hard 版本id
        版本id只需前几位,只要能唯一标记某个版本即可
        该命令即可回退也可前进

4.修改撤销：
    1.丢弃工作区修改：
        1.手动删除
        2.用版本库最新版本覆盖本地文件：git checkout -- 文件/目录
            上述命令本质上是用版本库最新版本覆盖本地工作区内容，可以用来实现本地工作区内容回退以及恢复本地误删文件；

    2.从暂存区回工作区：
        方式1：
            git restore --staged 文件/目录
        
        方式2：
            git reset HEAD 文件/目录
    
    3.从版本库撤销：
        直接回退到上一个版本即可

5.版本删除：
    删除版本库中指定版本：git rm 文件/目录
